5. Failed Pre-commit Hook (Husky)
The pre-commit hook, configured using Husky, automatically runs before every commit.
It executes:
Linting checks (ESLint + Prettier)
Validation script (validate.sh)
If either check fails, the commit is rejected.
This ensures that bad formatting, missing files, or invalid configuration cannot be committed to the repository.
